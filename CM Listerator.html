Welcome to the OpsTech IT CM Listerator



To begin, select a Review Option above.



Here is a brief explanation of each option's purpose:

  • LEVEL 1 TEAM REVIEW - A group of searches to show the pending queues of regional Level 1 CM approval teams.
  • CM BAR RAISER REVIEW - Usable by the different Bar Raiser teams to see their CM queue.
  • CODE FREEZE REVIEW* - Usable by the different "Code Freeze"/"Grey Day"/"Black Day" CM approvers to identify pending CMs as well as review past Unsuccessful CMs.
  • WILDCARD REVIEW* - Searches for all CMs that contain the provided user/team alias as an approver, filtered by selectable CM statuses.

* - These options utilize the date window on the left as the basis for their searches. Default window is 24 hours.




LEVEL 1 APPROVER TEAM REVIEW

Queue Search Functions to support Level 1 CM Approver Teams.

Enter the Team Identifier of a Level 1 Approver Team in the text box below, then select the button for the desired queue.

opstechit- -cml1



The search buttons can be used to change how far in advance the approval queue will seek:

  • Regular Queue - This is the daily work queue for Level 1 Teams. This search shows all CMs pending approval scheduled in the next three days.
  • Full Queue - This search shows all CMs pending Level 1 Team review, regardless of scheduled start.
  • X Days Ahead Queue - This search shows all CMs pending Level 1 Team review, using the Days In Advance field to the left.
  • Date Window Queue - This search shows all CMs pending Level 1 Team review, using the Date Window fields to the left.


LEVEL 1 APPROVER TEAM WEEKLY QUALITY CHECK

Search functions to support Team Leaders in performing weekly quality checks for their CML1 team.

Enter the Team Identifier of a Level 1 Approver Team in the text box below, then select the button for the desired quality check function.

opstechit- -cml1




The search buttons can be used to select what type of CMs will be shown for the quality check:

  • Unsuccessful CMs - This search shows all CMs approved by the CML1 team that are closed as "Unsuccessful" in the last week
  • Off-Script CMs - This search shows all CMs approved by the CML1 team that are closed as "Off-Script" in the last week
  • Incomplete CMs - This search shows all CMs approved by the CML1 team in the last week that were closed with the "Cancelled" or "Aborted" statuses
  • Completed CMs - This search shows all CMs approved by the CML1 team in the last week that were completed
  • The "7 Days" search function is used for the standard Weekly Quality Check. The "Date Window" search function can be used for larger-scale reviews based on the left Date Window fields.



CM BAR RAISER REVIEW

Search options used to view the queues for each CM Bar Raiser Team.

Select a CM Bar Raiser Team from the options below, then click one of the search buttons. Use the date window on the left to modify the search range.

NA EU UK IN APAC IO [ITRSS] IO [OpsTechIT-L3]



These two searches are used to view the daily queue and overall queue for each BR Team:

  • Regular Queue - Filtered - This shows CMs that are pending BR review, scheduled in the next 24 hours, where there is approval from at least one Level 1 Approver and one Level 2 Approver.
  • Regular Queue - Unfiltered - This shows all CMs pending BR review, scheduled in the next 24 hours, regardless of prerequisite approver statuses.
  • X Days Ahead Queue - This shows CMs pending BR review, filtered for prerequisite approvers in place, scheduled based on the left Days in Advance field.
  • Date Window Queue - This shows CMs pending BR review, filtered for prerequissite approvers in place, scheduled based on the left Date Window fields.




CODE FREEZE REVIEW

Searches used for finding CMs reviewable by the various Code Freeze Restricted Day and Blocked Day Approvers.

Select one of the approvers from the options below, sorted by Day Type role, then click one of the search buttons. Use the Date Window on the left to modify the search range.

Restricted Day Approvers:

opstechit-na-l7 opstechit-emea-l7 opstechit-in-l7 opstechit-apac-l7 opstechit-core-l7

Blocked Day Approvers:

readgleq manf kishorel-IN kishorel-APAC jeffreya



The Code Freeze search has two options for queue management and two options for identifying past Unsuccessful CMs:

  • Call Review - Filtered - This shows CMs pending Code Freeze review, scheduled using the left Date Window, where there is approval from at least one Level 1 Approver, one Level 2 Approver, and one Level 3 Approver.
  • Call Review - Unfiltered - This shows all CMs pending Code Freeze review, scheduled using the left Date Window, regardless of prerequisite approver statuses.
  • Failed CMs - 24 Hours - This search looks for all Unsuccessful CMs approved by the selected Code Freeze approve within the last 24 hours.
  • Failed CMs - Weekend - This search looks for all Unsuccessful CMs approved by the selected Code Freeze approver within the last three days. Best for reviewing on Mondays.

For more information on the OpsTech IT Code Freeze process, view the Policy document [here].




WILDCARD REVIEW

An open search for a provided user/alias filtered by Approval need, CM status, and date window

Enter an approver login or alias, select at least one CM status, then click one of the search buttons. Use the date window on the left to modify the search range.



Select the CM Statuses this search should check for:

SELECT/DESELECT All | Draft/Preflight Pending Approval Scheduled In Progress Sent For Rework Rejected Paused/Aborted Cancelled Completed


This search has two main functions, further divided by date-based filters:
  • Pending Approval - X Days Ahead - Searches CMs where the provided user/alias is Pending Approval, filtered by selected CM statuses and the left Days in Advance field
  • Pending Approval - Date Window - Searches CMs where the provided user/alias is Pending Approval, filtered by selected CM statuses and the left Date Window fields
  • All CMs - X Days Ahead - Searches all CMs where the provided user/alias is on the approval chain, regardless of approval status, filtered by selected CM statuses and the left Days in Advance field
  • All CMs - Date Window - Searches all CMs where the provided user/alias is on the approval chain, regardless of approval status, filtered by selected CM statuses and the left Date Window fields


ADDITIONAL SEARCHES

- A search to find all future CMs where the provided WHID is in the CM's "Fulfillment Center" field.
- A search to find all future CMs where the CM Technician matches the provided username.
- A search that opens the Phone Tool page for the provided user, or a Phone Tool search if there are multiple potential matches.
